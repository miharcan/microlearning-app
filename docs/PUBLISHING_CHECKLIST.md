# Public Publishing Checklist

Use this checklist before each push.

## Content Safety
1. Confirm no private code from product repos was copied.
2. Confirm no full content packs or large data artifacts were added.
3. Confirm no proprietary thresholds/policies are exposed.
4. Confirm screenshots do not show private user data.

## Security
1. Search for secrets: API keys, tokens, passwords, signing references.
2. Verify no local paths or private infra identifiers remain.

## Positioning Quality
1. README clearly explains product value.
2. Screenshots are current and high quality.
3. IP boundary is still accurate.
4. Public summary aligns with current product direction.
